## 1. 书管理

玩家"异常特性"法术书丢失、被其他玩家误拿、未正确绑定时使用。

### 1.1 `/magic givebook <player> [forceRebuild]`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 参数 | `player` = 在线玩家；`forceRebuild` = 0/1，缺省 0 |
| 作用 | 给玩家发一本异常法术书，并绑定到 Curios 书槽 |

- `forceRebuild=0`：幂等操作。如果玩家已经有合法书就仅校验绑定，不重建。
- `forceRebuild=1`：强制删除玩家身上所有匹配 `boundBookId` 的旧书，按 `LastValidBookSnapshot` 重建一本新的。

**典型场景**：新玩家首次进服未自动获得；管理员拍板给某位玩家"重开号"。

```
/magic givebook Steve          # 正常发放/确认
/magic givebook Steve 1        # 强制重建（旧书会被清除）
```

### 1.2 `/magic forceequip <player>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 作用 | 把玩家已有的异常书**强制放回 Curios 书槽**，推送到客户端 |

**典型场景**：玩家报告"书槽是空的但物品栏里有书"；`/magic givebook` 其实已经内置了这个动作，此命令只在已知 book id 正确、只差装备动作时使用。

```
/magic forceequip Steve
```

### 1.3 `/magic fixbook <player>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 作用 | 自动定位、绑定、补绑、放回书槽；**是 givebook + forceequip 的合并快捷版** |

**典型场景**：玩家报"我的异常书不见了 / 书上没有我的名字 / 书装在副手但没进槽"时，先跑这条。

```
/magic fixbook Steve
```

## 2. 法术管理

对玩家异常书里的法术做增删改查。注意**所有 `<spell>` 参数都支持三种写法**：

- 注册 ID：`sonic_sense`
- 中文名：`音波`
- 完整 ID：`corpse_campus:sonic_sense`

### 2.1 `/magic add <player> <spell> <level> <count>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 参数 | `level` 1–10；`count` 1–64 |
| 作用 | 向目标书写入指定法术 |

行为细节：
- 同名法术已存在 且等级更低 → **升级**到 `level`
- 同名法术已存在 且等级 ≥ `level` → **跳过**（不报错但返回 fail）
- 书槽已满（上限 35）→ 报错
- `spell=endless_life` 且封锁开启 → 拒绝并提示用 `/magic seal endless_life false` 解封

**典型场景**：
```
# 一键给 Steve 装音波 Lv.1
/magic add Steve 音波 1 1

# 升级到 Lv.5
/magic add Steve sonic_sense 5 1

# 批量尝试 3 次（失败也不重复占槽）
/magic add Steve elementalist 1 3
```

### 2.2 `/magic spells <player>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 作用 | 只读列出目标书内全部装载法术：ID / 等级 / 阶级 |

```
/magic spells Steve
```

### 2.3 `/magic clear <player> <spell>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 作用 | **完整移除**指定法术，不留任何等级 |

```
/magic clear Steve 印记
```

### 2.4 `/magic clearall <player>`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 作用 | 清空目标书内**全部**法术（保留觉醒/序列/阶位字段） |

**典型场景**：玩家需要洗点；或作弊数据清理。**注意**这条不会重置 `AnomalyMainSequence`，如果要整体"打回未觉醒"请用 `/magic unawaken`。

```
/magic clearall Steve
```

### 2.5 `/magic remove <player> <spell> [count]`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |
| 参数 | `count` 1–64，缺省 1 |
| 作用 | **扣减**指定法术的等级 `count` 次；等级降到 0 自动移除 |

**典型场景**：精细化微调等级。例如玩家通过剧情赠与拿到了多余等级，要降一档。

```
/magic remove Steve healing          # 扣 1 级
/magic remove Steve healing 3        # 扣 3 级
```

## 3. 觉醒状态

直接改书上的 `AnomalyAwakened / AnomalyMainSequence / AnomalyHighestRank` 三个核心字段。

### 3.1 `/magic unawaken <player>`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 作用 | 彻底取消玩家觉醒：清空书内法术 + 清主序列 + 清最高阶 + 从 cap 集合移除 |

**典型场景**：玩家作弊/误给/主动请求重置。此命令会释放上限名额，其他玩家可以重新进入 cap。

```
/magic unawaken Steve
```

### 3.2 `/magic setsequence <player> <sequence>`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 参数 | `sequence` ∈ `xujing/rizhao/dongyue/yuzhe/shengqi` 或 `虚境/日兆/东岳/愚者/圣祈`；传 `none`/`无` 清除 |
| 作用 | **只改主序列字段**，不动书内法术与最高阶 |

```
/magic setsequence Steve 虚境
/magic setsequence Steve none        # 清除序列绑定
```

### 3.3 `/magic setrank <player> <rank>`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 参数 | `rank` ∈ `B/A/S/NONE` 或 `一级/二级/三级/无` |
| 作用 | **只改最高阶字段**，不动书内法术与主序列 |

```
/magic setrank Steve A
/magic setrank Steve 无             # 相当于未觉醒态
```

## 8. 上限系统

### 8.1 `/magic limit info`

| 字段 | 值 |
| --- | --- |
| 权限 | 2 |

/magic limit info
```

### 8.2 `/magic limit set <value>`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 参数 | `value` 1–9999 |
| 作用 | 运行时修改上限，立即生效并持久化 |

```
/magic limit set 50
```

### 8.3 `/magic limit enable` / `/magic limit disable`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 作用 | 开启 / 关闭整个上限检测（关闭后 B 级异常特性正常掉落） |

### 8.4 `/magic limit recount`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |

**典型场景**：数据不一致（例如直接编辑存档后）；或者批量 `unawaken` 之后确认名额释放情况。

```

## 9. 生生不息封锁

`endless_life`（圣祈 S）一旦被任何玩家**施放成功**就会全局封锁，任何人再也无法获得。用以下命令可手动查询或解封。

### 9.1 `/magic seal endless_life [sealed]`

| 字段 | 值 |
| --- | --- |
| 权限 | **4** |
| 参数 | `sealed` = `true` / `false`，**不传则查状态** |
| 作用 | 查看或手动设置【生生不息】的全局封锁状态 |

**典型场景**：
- 运营失误（某位玩家违规获得并施放了生生不息），需要重新开放 → `/magic seal endless_life false`
- 剧情需要，提前锁死某一阶段 → `/magic seal endless_life true`

```
/magic seal endless_life                # 查状态
/magic seal endless_life true           # 强制封锁
/magic seal endless_life false          # 解除封锁
```

## 10. 回溯之虫镜像备份

为虚境 S 级 `rewind_worm` 服务。三阶段串行扫描：方块态 → 方块实体 → 生物实体。**三阶段都跑完后**玩家才能施放 `rewind_worm`。

### 10.1 `/magic rewind backup create <player> [radius]`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 参数 | `player` = 扫描中心锚点所在玩家；`radius` 16–10000 格，缺省 3000 |
| 作用 | 以玩家当前坐标为中心，启动三阶段异步备份扫描 |

**每 tick 处理预算**：
- Phase 1 方块态：2 chunks/tick
- Phase 2 方块实体：4 chunks/tick
- Phase 3 生物实体：4 chunks/tick

```
/magic rewind backup create Steve             # 默认半径 3000 格
/magic rewind backup create Steve 500         # 调试小范围
/magic rewind backup create Steve 1500        # 中型地图
```

### 10.2 `/magic rewind backup status`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 作用 | 查看当前扫描阶段、每阶段已完成 chunks 数、当前阶段百分比、预计剩余时间、镜像维度加载情况 |

```
/magic rewind backup status
```

### 10.3 `/magic rewind backup cancel`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 作用 | 中止当前扫描。**已复制进镜像的方块/实体不会被清除**；下次 `create` 会覆盖。 |

```
/magic rewind backup cancel
```

### 10.4 回溯之虫标准运营流程

```
 ┌─────────────────────────────────────────────────────────────┐
 │ Step 1. 确认镜像维度已加载（旧存档需一次 /execute in ...）    │
 ├─────────────────────────────────────────────────────────────┤
 │ Step 2. /magic add <玩家> rewind_worm 1 1                    │
 ├─────────────────────────────────────────────────────────────┤
 │ Step 3. /magic rewind backup create <玩家> [半径]            │
 │         → 进入 Phase 1 SCAN_BLOCKS                           │
 ├─────────────────────────────────────────────────────────────┤
 │ Step 4. 轮询 /magic rewind backup status 直到显示 READY       │
 │         → Phase 1 → Phase 2 → Phase 3 → READY                │
 │         （3000 格半径约需 4–6 小时）                          │
 ├─────────────────────────────────────────────────────────────┤
 │ Step 5. 通知玩家施放 rewind_worm（去程）                      │
 │         → 玩家穿越到镜像维度，原坐标同步                      │
 ├─────────────────────────────────────────────────────────────┤
 │ Step 6. 等 30 分钟共享冷却走完                                │
 │         （由 ISS 法术系统统一管理，无独立 NBT 冷却）          │
 │         → 玩家再次施放 rewind_worm（回程）                    │
 │         → 回到锚点维度 + 原坐标 + 原视角                      │
 └─────────────────────────────────────────────────────────────┘
```

> **重要变更（2026-04-23）**：`RewindWormRuntime` 不再维护任何 NBT 冷却字段，
> 30 分钟冷却完全由 `RewindWormSpell.defaultConfig.setCooldownSeconds(1800)` 触发的
> ISS 法术冷却管理。任何与"回溯之虫剩余冷却 / 内置 CD 检测"相关的旧文档行为已下线。

## 11. 每日限定法术刷新

> **背景**：早期版本通过 `level.getDayTime() / 24000L` 自动检测"游戏日"实现"日轮金乌每日 1 次"。
> 自 2026-04-23 起：**取消所有自动按天检测**，改为"每次施放即锁定"，必须管理员显式刷新才能再次使用。
> 实现机制：维护一个全局 `generation`（`DailyAbilityRefreshState` SavedData）；玩家每次施法把当前 generation 写入自身 NBT；管理员调用刷新指令让 generation++，所有 generation 落后于全局值的玩家（含离线）下次施法即被放行。

**当前覆盖的法术**：

| 序列 | 阶 | 法术 | 锁定语义 |
| --- | --- | --- | --- |
| 日兆 | S | `golden_crow_sun`（日轮金乌） | 每次召唤即锁定，提示"日轮金乌已使用过，需管理员执行 /magic refresh all" |

后续若有新增"每日 / 每周 / 每剧情节点"语义的法术，建议接入同一个 `DailyAbilityRefreshState`，沿用 `TAG_*_USED_GEN` 模式即可，无需另写定时检测。

### 11.1 `/magic refresh all`

| 字段 | 值 |
| --- | --- |
| 权限 | **3** |
| 作用 | 让全局 `generation++`；所有曾用过覆盖范围内法术的玩家（含离线）下一次施法都会被自动放行 |

- **不需要指定玩家**。一次刷新等价于"全服重置"。
- **离线玩家也会被覆盖**——因为判定基于"玩家记录的 gen 与全局 gen 的比较"，玩家上线后施法时才比较。
- 当前 generation 可在 [`/magic config`](#43-magic-config) 输出末尾的 `每日限定法术 generation` 一行查看。

```
/magic refresh all
```

**典型场景**：
- 每日服务器维护后开放新一轮日轮金乌；
- 战斗活动开始前统一解锁日轮，确保选手起步公平；
- 误用导致玩家锁定，需要单点放行（目前仅有"全服一刀切"指令；如需细化到玩家请提 issue）。

## 附录 B · 常见运营场景脚本

### B.1 新开服初始化

```
# 1. 确认上限与封锁状态
/magic config
/magic limit info

# 2. 若需要调整上限
/magic limit set 40
/magic limit enable

# 3. 给服主 & 管理员自测账号发书
/magic givebook Admin
/magic state Admin

# 4.（可选）先把镜像维度触发加载（方便后续 rewind 备份不会 MIRROR_MISSING）
/execute in corpse_campus:rewind_mirror run tp @s 0 80 0
```

### B.2 给玩家发放指定序列（剧情 / 补偿）

```
# 1. 给虚境 B 级吞噬物（2 个备用）
/magic trait give Steve 虚境 B 2

# 2. 或者直接写入法术 + 设序列/阶位（绕过吞噬物）
/magic add Steve sonic_sense 1 1
/magic setsequence Steve 虚境
/magic setrank Steve B

# 3. 验证
/magic state Steve
/magic who
```

### B.3 玩家报"异常书丢了 / 不能用"

```
# 先一键修复
/magic fixbook Steve

# 查一下状态
/magic state Steve
/magic spells Steve

# 如果仍然不对，导出 NBT 给开发
/magic dump Steve

# 最后手段：强制重建
/magic givebook Steve 1
```

### B.4 给 S 级玩家做镜像备份（rewind_worm 上线前）

```
# 1. 先给法术
/magic add Steve rewind_worm 1 1

# 2. 让 Steve 站在备份锚点（通常主城 / 新手村 / 争夺点）

# 3. 启动扫描（先用小半径测试链路）
/magic rewind backup create Steve 500

# 4. 每隔 15 分钟轮询一次
/magic rewind backup status

# 5. 确认 Phase 1/2/3 全部 READY 后，通知玩家可以施放

# 6. 若要换锚点，先取消再重来（会覆盖镜像维度内容）
/magic rewind backup cancel
/magic rewind backup create Steve 3000
```

### B.5 玩家作弊 / 误给，整体回滚

```
# 1. 取消觉醒（清空法术 + 释放名额）
/magic unawaken Steve

# 2. 重算全服上限
/magic limit recount

# 3. 如果涉及生生不息误封锁，解封
/magic seal endless_life false

# 4. 验证
/magic state Steve
/magic who
/magic limit info
```

### B.6 每日维护：刷新日轮金乌使用次数

```
# 1.（可选）查看当前 generation
/magic config

# 2. 全服一键放行所有"每次施放即锁"的法术（目前仅日轮金乌）
/magic refresh all

# 3. 验证 generation 已 +1
/magic config
```

> 该指令不需要指定玩家；离线玩家也会被覆盖（依赖玩家自身 NBT 中记录的 generation 比较）。
