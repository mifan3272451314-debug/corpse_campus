# 音波 / 嗅觉渲染问题调查报告

## 一、问题概述

当前 [`音波`](src/main/java/com/mifan/spell/SonicSenseClientHandler.java) 与 [`嗅觉`](src/main/java/com/mifan/spell/OlfactionClientHandler.java) 两个客户端可视化能力仍未达到可用状态。

截至本报告编写时，玩家侧实际反馈如下：

- **音波**：
  - 已经可以在部分场景中做到“隔墙看到目标”；
  - 但标记位置仍然存在**明显错位**；
  - 并且当前表现疑似存在**前后方向判断异常**，即“生物在背后时反而更正常”。
- **嗅觉**：
  - 玩家反馈仍然是**完全看不到任何有效显示**；
  - 现有实现即便已经多次调整，仍未形成稳定可见的脚印 / 轨迹表现。

## 三、音波问题调查结论

### 1. 已确认现象

- 音波改成 HUD 投影方案后，确实更容易实现“隔墙显示”；
- 但 HUD 投影链路并不天然等于“贴在实体身上”；
- 玩家实测发现方向表现仍有异常，怀疑当前投影坐标或前后判断仍有问题；
- 当前版本已经移除了早先基于 [`screenPos.z`](src/main/java/com/mifan/spell/SonicSenseClientHandler.java) 的过滤，但从结果看，**问题并没有被彻底消除**。

### 2. 已尝试过的修复方向

- 将音波从世界空间 billboard 改为 HUD 投影；
- 恢复 [`worldToScreen()`](src/main/java/com/mifan/spell/SonicSenseClientHandler.java) 的投影路径；
- 改用插值坐标与 GUI 缩放后的尺寸；
- 调整实体锚点高度；
- 去掉基于 `z` 范围的显示过滤。

### 3. 当前判断

音波问题尚未真正解决，原因大概率还在以下几类之一：

1. [`worldToScreen()`](src/main/java/com/mifan/spell/SonicSenseClientHandler.java) 的相机空间换算与当前 Forge / 1.20.1 渲染链并不完全一致；
2. 仅依赖 HUD 投影时，实体的“世界位置”与“玩家感知中的模型位置”仍有差距，尤其在坡地、半格高度、移动插值和镜头快速转动时会被放大；
3. 当前前后关系判断仍有隐藏问题，可能并不是单纯的 `z` 过滤，而是投影矩阵、旋转四元数或屏幕空间映射方向上的偏差。

### 4. 当前结论

> **音波目前仍属于“隔墙可部分显示，但定位错误 / 错位未修复”的状态，不能视为完成。**

## 四、嗅觉问题调查结论

### 1. 已确认现象

- 玩家反馈为“仍然看不到任何东西”；
- 之前的地面四边形脚印方案，实际效果不可见或极不稳定；
- 后续改为粒子脚印轨迹方案后，玩家仍然反馈为不可见。

### 2. 已尝试过的修复方向

- 提高脚印贴片高度；
- 关闭深度测试 / 深度写入；
- 增强贴片颜色与透明度；
- 增加红色 [`DustParticleOptions`](src/main/java/com/mifan/spell/OlfactionClientHandler.java) 粒子；
- 最后进一步整体重写为“纯粒子脚印轨迹”方案。

### 3. 当前判断

嗅觉当前最大问题不是“颜色不够亮”，而是**整条可视化链路没有被验证跑通**。潜在原因包括：

1. [`tick()`](src/main/java/com/mifan/spell/OlfactionClientHandler.java) 虽然在客户端执行，但目标筛选条件可能导致根本没有捕获到有效实体；
2. “低于 75% 血量”这一条件在测试场景里可能并未稳定满足；
3. 客户端自己推导脚印，而没有直接消费服务端在 [`AbilityEventHandler.updateOlfactionTrail()`](src/main/java/com/mifan/spell/AbilityEventHandler.java) 中维护的轨迹数据，导致显示逻辑与真实运行时状态脱节；
4. 粒子虽然调用了 [`ClientLevel.addParticle()`](src/main/java/com/mifan/spell/OlfactionClientHandler.java)，但仍可能因为触发条件、时机或位置计算错误，导致肉眼看不到。

### 4. 当前结论

> **嗅觉目前仍属于“功能未验证跑通，玩家视角近似完全无显示”的状态，不能视为完成。**

## 五、与外部参考实现的对比结论

已参考外部项目中“侦探听音定位”相关代码，核心结论如下：

- 参考实现的“隔墙显示”依赖的是 **HUD 投影**，而不是世界空间贴片；
- 参考实现显示的是**声音事件位置**，不是持续绑定在实体模型上的 3D 标记；
- 因此它天然更容易做到“隔墙可见”，但并不等于可以直接无损套用到当前音波 / 嗅觉设计。

这意味着：

- **音波**如果继续走 HUD 路线，就必须单独解决“贴合实体位置”的投影精度问题；
- **嗅觉**如果继续走脚印路线，则更应优先保证“先稳定看见”，再去追求更复杂的地面印记造型。
