>   [`EvolutionRitualService`](src/main/java/com/mifan/anomaly/EvolutionRitualService.java) /

## 1. 背景与定位

### 1.1 为什么加这条通道

A 级觉醒此前只有一条路径：吞噬 A 级吞噬物 → 右键使用。过于依赖掉落，玩家无法通过**玩法行为**
主动进入 A 级。进化觉醒是一条平行通道：玩家先搭祭坛 → 消耗 B 级特性 + 仪式配方生成**胚胎** →
背包持胚胎期间达成各自独特的触发条件 → 胚胎原位转化为**核心** → 右键核心装进异常书。

### 1.2 定位
- **只在 A 级层触发**。B 级仍走 [自然觉醒.md](./自然觉醒.md)；S 级保持现有路径不变
- **与吞噬 A 级吞噬物互斥**：已 A 级玩家不再走进化通道（仪式祭坛直接拒绝产出胚胎）
- **不改 40 上限本身**：进化是"B → A 升位"，不新增席位
- **非目标**：不改 `AnomalyBookService` 原有装书逻辑、不动 ISS 原生 `SpellBook`

## 2. 系统总览

### 2.1 两条 A 级通道对比

| 维度 | 吞噬 A 通道（现有） | 进化觉醒通道（本文档） |
| --- | --- | --- |
| 入口 | 右键 A 级吞噬物 `ItemStack` | 祭坛融合 → 胚胎 → 条件 → 核心 → 吞下 |
| 消耗物品 | 是（物品 shrink(1)） | 两次：仪式物品 + 核心（shrink(1)） |
| 法术选择 | 从该流派 A 级池随机抽 1 个 | **定向**：每条配方绑定 1 个 A 级 spellId |
| 前置 | 已 B 级 + 主序列匹配 | 同上 + 已搭好祭坛 + 达成条件 |
| 耗时 | 秒级 | 分钟到小时级（取决于条件） |
| 复杂度 | 低（单点物品） | 高（祭坛结构 + 玩法条件） |

### 2.2 用户锁定的核心口径（2026-04-25）

| 口径 | 决定 |
| --- | --- |
| 流程 | 祭坛 → 胚胎 → 条件 → 核心 → 右键吞下（两段式） |
| 多胚胎并存 | 无限制（含跨流派）；同 tick 只处理背包顺序靠前的第一枚 |
| 已 A 级玩家 | 禁用进化通道（仪式直接拒绝产出胚胎） |
| "其他途径异能者" | `MainSequence ≠ 施法者` 且 对方 `AnomalyAwakened = true` |
| 玩家尸体配料 | 祭坛半径吸收 `CorpseEntity`（Corpse mod 的实体） |
| 死亡重置 | 仅清 `EvolutionProgress` NBT；胚胎 / 核心物品**不清**，按普通物品掉落规则 |

## 3. 祭坛结构（5 流派）

> 视觉拓扑：`center` 是被 shift+右键的中心方块，自身位于"底座"层；四角装饰位于 `center.above()`。
> ```
>          [corner] [ ]   [corner]
>          [   ]  [pillar]  [   ]    ← center.above() 层
>          [corner] [ ]   [corner]
>
>          [floor][floor][floor]
>          [floor][center][floor]    ← center 层（底座 + 中心方块）
>          [floor][floor][floor]
> ```

| 流派 | 底座方块 | 中心方块 | 四角方块 |
| --- | --- | --- | --- |
| 虚境 | `minecraft:gilded_blackstone` | `minecraft:enchanting_table` | `minecraft:black_candle` |
| 日兆 | `minecraft:moss_block` | `minecraft:flowering_azalea` | `minecraft:sunflower` |
| 东岳 | `minecraft:obsidian` | `minecraft:crying_obsidian` | `minecraft:soul_lantern` |
| 愚者 | `minecraft:amethyst_block` | `minecraft:lodestone` | `minecraft:skeleton_skull` |
| 圣祈 | `minecraft:sculk` | `minecraft:beacon` | `minecraft:sculk_sensor` |

校验：[`EvolutionAltarStructure#matches`](src/main/java/com/mifan/anomaly/EvolutionAltarStructure.java)。

## 4. 10 条仪式配方 + 觉醒条件

> **仪式触发**：玩家 shift+右键中心方块 → [`EvolutionRitualService#attemptFusion`](src/main/java/com/mifan/anomaly/EvolutionRitualService.java)。
> 物品"丢进祭坛"= `ItemEntity` 落在 center 正上方 AABB `inflate(1.2, 2.0, 1.2)` 内。
> "玩家尸体"= `CorpseEntity` 在半径 5 格内（Corpse mod 未加载时视为 0 具）。
> **前置** = 玩家已 B 级觉醒 + 主序列等于该祭坛流派 + 最高阶 == B（已 A 级拒绝）。

## 5. 数据结构

### 5.1 玩家持久化 NBT

挂在 `player.getPersistentData()[Player.PERSISTED_NBT_TAG] → AnomalyP0.EvolutionProgress` 下：

```
EvolutionProgress:
  recorder_officer:
    cast_ts: [long, long, ...]         // 施法时间戳滑窗
    cast_actor_most/least: [long, ...] // 施法者 UUID 滑窗（most/least bits 各一份，等长）
  elementalist:
    flame_seen: boolean
    drown_seen: boolean
    lightning_seen: boolean
  light_prayer:
    continuous_light_tick: long
  impermanence_monk:
    continuous_crouch_tick: long
  executioner:
    player_kill_count: int
  life_thief:
    attack_ts / attack_actor_most / attack_actor_least: [long, ...]  // 同记录官结构
  // 其余（模仿者 / 点金客 / 嫁接师 / 摆渡人）为瞬时条件，无累计字段
```

### 5.2 物品形态转换

| 形态 | Item | 产出路径 | 消费路径 |
| --- | --- | --- | --- |
| 胚胎 | [`SpellEmbryoItem`](src/main/java/com/mifan/item/SpellEmbryoItem.java) + NBT `EmbryoSpellId` | 祭坛融合 | 被动凭证，不主动消耗 |
| 核心 | [`EvolutionCoreItem`](src/main/java/com/mifan/item/EvolutionCoreItem.java) + NBT `EvolutionCoreSpellId` | 条件达成时 `transformEmbryoToCore` 原位替换胚胎 | 右键 → `AnomalyBookService#applyScrollSpell` → shrink(1) |

## 6. 40 席位与上限行为

进化觉醒是"B → A"升位，**不新增 40 席位**。玩家已被记入 `AnomalyLimitService` 的觉醒集合，
进化通道仅调用 `applyScrollSpell`（已觉醒 + 主序列匹配分支），不会二次触发 `markAwakened`。
满员 40 的拦截对进化通道无效（因为本来就不占新席位）。

## 7. 死亡重置

### 7.1 现有行为（沿用）
[`AnomalyEventHandler#onPlayerDeath`](src/main/java/com/mifan/anomaly/AnomalyEventHandler.java) 已处理：
- 清空书内法术、清序列绑定
- 释放 40 席位
- 生成历史阶段物品掉落

### 7.2 进化觉醒追加的清理
在同一钩子里追加（与 `NaturalAwakeningService.clearProgress` 并列）：
- `EvolutionAwakeningService.clearProgress(player)` —— 清 `EvolutionProgress` NBT

### 7.3 **不清** 胚胎 / 核心物品（用户口径 #6）
- 胚胎 / 核心作为普通物品，受 `keepInventory` gamerule 与 `LivingDropsEvent` 常规流程影响
- 若玩家跨越死亡保留了胚胎，进度归零后仍可重新累计；核心保留则可任意时刻吞下（前提仍是主序列匹配且未 A 级）

## 8. 客户端反馈

### 8.1 祭坛触发
- 成功：`message.corpse_campus.evolution_embryo_born`（金色）+ `BEACON_ACTIVATE` 音效 + END_ROD 粒子
- 结构残缺：`message.corpse_campus.evolution_altar_incomplete`（红色）
- 主序列不符：`message.corpse_campus.evolution_denied.wrong_sequence`（红色）
- 已 A 级：`message.corpse_campus.evolution_denied.already_a`（红色）
- 未觉醒：`message.corpse_campus.evolution_denied.not_awakened`（红色）
- 材料不匹配：`message.corpse_campus.evolution_recipe_not_matched`（黄色）

### 8.2 胚胎→核心转化
- `message.corpse_campus.evolution_core_forged`（金色）+ `BEACON_POWER_SELECT` 音效
- 转化瞬时、原位（同一 inventory slot），不走掉落

### 8.3 核心右键
- 复用 `AnomalyBookService#applyScrollSpell` 返回的 `AbsorbResult.message`
- 成功走 `message.corpse_campus.scroll_spell_granted`

## 9. 待扩展 / 限制

1. **独立贴图**：核心当前复用胚胎贴图（5 张流派色），Phase 7 仅接入 JSON override 骨架；后续可出 5 张 `core_<school>.png`
2. **进度查询指令**：`/magic evolution progress [player]` 暂未实现；当前依赖 NBT 编辑器查看
3. **玩家尸体配料语义**：`CorpseEntity` 无 item 形式，必须**拖/带**到祭坛 5 格内；跨维度不可行。
   消耗尸体时通过 [`EvolutionCorpseBridge#dropContentsAndDiscard`](src/main/java/com/mifan/anomaly/EvolutionCorpseBridge.java) 把死者的主物品栏 / 装备 / 副手 / 额外物品反射读出后生成 `ItemEntity` 落在祭坛原位，再 `entity.discard()` —— **不会吞掉玩家的装备**。
4. **Corpse mod 缺失**：`EvolutionCorpseBridge.CORPSE_ENTITY_CLASS` 为 null → 3 条需要尸体的配方（无常僧 / 嫁接师 / 摆渡人）永远无法满足 + 摆渡人觉醒条件永远不触发
5. **多胚胎并存**：`transformEmbryoToCore` 按 `listEmbryos` 的 hotbar-main-offhand 顺序找首枚匹配；并发 tick 的多条件同时达成只处理第一枚，其余留到下一 tick
