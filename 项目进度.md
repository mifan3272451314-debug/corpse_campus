## 已完成内容（按阶段）

### 1. 前置开发环境已接入

- 本地 `libs/` 目录：Iron's Spells 'n Spellbooks / Curios API / GeckoLib / Player Animation Lib 全部就位
- `build.gradle` 通过 `flatDir` + `fg.deobf(...)` 引入
- `mods.toml` 已声明运行依赖

### 2. 五大自定义流派已注册

在 [`ModSchools.java`](src/main/java/com/mifan/registry/ModSchools.java) 下注册：`xujing / rizhao / dongyue / yuzhe / shengqi`。每个流派具备资源路径、显示名称、颜色、施法音效映射、伤害类型映射、focus 标签入口。

### 3. 本地化与资源

- `zh_cn.json` / `en_us.json` 流派名称、法术名、tooltip、错误码、OP 通知等全部键齐全（语言键约 300+ 条）
- 33 个法术图标齐全；3 个 S 级图标齐全（rewind / great_necromancer 等仍按模组默认样式走可，不阻塞）

### 4. 异常法术书 P0 闭环

入口：[`AnomalyBookService`](src/main/java/com/mifan/anomaly/AnomalyBookService.java) + [`AnomalyEventHandler`](src/main/java/com/mifan/anomaly/AnomalyEventHandler.java)。

- 异常书物品注册 + Curios `spellbook` 槽绑定
- 首次登录 / 重生 / 维度切换时 `ensureBookPresent` 自动补书 + 强制佩戴
- 书绑定主人 UUID，非主人拾取后进入物品栏自毁
- 法力加成 + 五流派强化按**已搭载法术的最高位阶**实时重算（B +10HP/+75 法力/+25%；A +20HP/+150/+50%；S +40HP/+250/+50%）
- 死亡清空：`clearBookAndCollectTraitDrops` 清空法术 + 清 `AnomalyAwakened / MainSequence / HighestRank` + 从 `AnomalyLimitService` 释放名额

### 5. 40 上限系统

- [`AnomalyLimitService`](src/main/java/com/mifan/anomaly/AnomalyLimitService.java) 为 `SavedData`（文件名 `corpse_campus_anomaly_limit.dat`）
- 默认 40 人上限，可通过 `/magic limit set` 在线调整
- 觉醒时 `markAwakened(uuid)`，死亡时 `clearAwakened(uuid)`
- **当前口径（2026-04-23 更新）**：40 上限**只拦觉醒通道，不拦发书、不拦掉落**。即使满员，B 级特性照常掉落；玩家吞噬时返回 `anomaly_cap_reached` 失败

### 6. 自然觉醒通道

[`NaturalAwakeningService`](src/main/java/com/mifan/anomaly/NaturalAwakeningService.java) + [`NaturalAwakeningEventHandler`](src/main/java/com/mifan/anomaly/NaturalAwakeningEventHandler.java)：

| 原子条件 | NBT key | 阈值 | 对应法术 |
| --- | --- | --- | --- |
| 累计放置告示牌 | `sign_place_count` | 100 | 印记 |
| 累计破坏方块 | `block_break_count` | 1000 | 耐力 |
| 累计开垦耕地 | `farmland_till_count` | 75 | 沃土 |
| 累计催熟作物 | `bonemeal_crop_count` | 100 | 宁禾 |
| 平砍次数 | `melee_attack_count` | 500 | 躁狂 |
| 被怪物攻击 | `hit_by_monster_count` | 200 | 本能 |
| 击杀怪物 | `monster_kill_count` | 200 | 冥化 |
| 一次性坠落 | `fall_burst_max` | 200 格 | 万象 |
| 累计接触墙 | `wall_touch_tick` | 6000 tick | 磁吸 |
| 连续蹲伏 | `crouch_tick` | 6000 tick | 危机 |
| 连续爬行 | `crawl_tick` | 2400 tick | 危机 |

- 觉醒者的进度全部冻结；满员时进度仍累加但不执行觉醒
- 死亡时 `clearProgress` 全量清零
- 触发觉醒后赠予 60 ticks 无敌窗口（`grantAwakeningProtection`）
