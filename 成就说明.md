## §2 Hook 机制（重点）

### §2.1 为什么需要 Hook

关键字段 `AnomalyAwakened` / `AnomalyMainSequence` / `AnomalyHighestRank` **不在玩家 PersistentData**，而是挂在**异常法术书 ItemStack 的 CompoundTag 上**（[AnomalyBookService.java:56-58](src/main/java/com/mifan/anomaly/AnomalyBookService.java#L56-L58)）。而法术书本身在 **Curios `spellbook` 饰品槽**，不在主背包。

原版 `minecraft:inventory_changed` 不扫描 Curios 槽；`nbt` predicate 走 `ForgeCaps."curios:inventory"` 深层路径极度脆弱且依赖 Curios 版本。**最可靠的方案**：用 Java 代码在状态变化瞬间直接 `advancement grant`。

### §2.2 Hook 类入口

[src/main/java/com/mifan/anomaly/AnomalyAdvancements.java](src/main/java/com/mifan/anomaly/AnomalyAdvancements.java) 暴露以下静态方法：

| 方法 | 触发哪些成就 |
| --- | --- |
| `onBookGranted(player)` | `root/root` |
| `onSequenceEntered(player, schoolPath)` | `sequence/<school>` |
| `onRankAdvancedToA(player, schoolPath)` | `advance/<school>` |
| `onRankAdvancedToS(player, schoolPath)` | `summit/<school>` |
| `onSpellLearned(player, spellId)` | `ability/<spell>` |
| `onNaturalCrawlReached(player)` | `natural/danger_sense_crawl` |
| `onDungeonCleared(player)` | `natural/dungeon` |
| `onEndlessLifeCast(player)` | `finale/endless_life_cast` |
| `onRewindMirrorEntered(player)` | `finale/rewind_mirror` |
| `onFortyCapReached(server)` | `finale/forty_cap`（全服广播） |
| `onGrafterCrossFour(player)` | `finale/grafter_cross` |
| `onIsolatedAscend(player)` | `finale/isolated_ascend` |

### §2.3 已注入的调用点

| 代码位置 | 触发内容 |
| --- | --- |
| `AnomalyBookService.applyAwakening` 末尾 | `root` + `sequence/<school>` + `ability/<spell>` + 按位阶 `advance/summit` |
| `AnomalyBookService` 卷轴通道 `writeHighestRank` 后 | `ability/<spell>` + `advance/summit`；**非圣祈登顶 S** 自动触发 `finale/isolated_ascend` |
| `AnomalyBookService.addSpell` 成功返回前 | `root` + `ability/<spell>`；**达到 40 上限**时对全服 broadcast `finale/forty_cap` |
| `AnomalyBookService.ensureBookPresent` 末尾 | `root`（覆盖首次发书 / 补发 / 登录修复） |

### §2.4 未注入的 Hook（待补）

以下 Hook 方法已声明但尚未在对应业务代码里调用，调用点建议见 §5：

- `onNaturalCrawlReached` — 爬行 120 秒达成时（[NaturalAwakeningEventHandler](src/main/java/com/mifan/anomaly/NaturalAwakeningEventHandler.java)）
- `onDungeonCleared` — 副本探索成功时（副本子系统未实装）
- `onEndlessLifeCast` — [EndlessLifeRuntime.notifyOperators](src/main/java/com/mifan/spell/runtime/EndlessLifeRuntime.java) 施放瞬间
- `onRewindMirrorEntered` — 回溯之虫镜像维度入口
- `onGrafterCrossFour` — [GrafterRuntime](src/main/java/com/mifan/spell/runtime/GrafterRuntime.java) 跨流派吸收 ≥4

## §6 Trigger 机制分类

### §6.1 `minecraft:impossible` + Java Hook（35 个）

JSON 层占位，代码层 `player.getAdvancements().award(advancement, criterionKey)` 发放。玩家感知与真实触发完全一致，但 `/advancement list` 显示 trigger 名称为 `impossible`。

### §6.2 `minecraft:tick` + `type_specific.player.stats`（30+ 个）

玩家每 tick 被检查，stat 累计值 ≥ 阈值时触发。**stat 永不回落**（除非 `/stats clear`）。

本项目用到的 stat 速查：

| type | stat | 单位 |
| --- | --- | --- |
| `custom` | `mob_kills` / `player_kills` / `deaths` / `jump` / `enchant_item` | 次数 |
| `custom` | `damage_dealt` / `damage_taken` | 1/10 点伤害 |
| `custom` | `fall_one_cm` / `climb_one_cm` / `walk_one_cm` | 厘米 |
| `custom` | `sneak_time` / `play_time` | tick |
| `custom` | `interact_with_*` / `sleep_in_bed` / `fish_caught` / `traded_with_villager` | 次数 |
| `used` | `<item>` | 使用次数（含右键放置） |
| `mined` | `<block>` | 挖方块次数 |

### §6.3 `minecraft:inventory_changed`（19 个）

"曾经持有过"即算达成。单物品或多物品 AND 组合（用 `requirements` 分组）。

### §6.4 其他专用 trigger

| trigger | 用途 |
| --- | --- |
| `changed_dimension` | 下界 / 末地入场 |
| `location` + `biomes` predicate | 生物群系踏足 |
| `player_killed_entity` | 击杀特定 boss |
| `entity_killed_player` | 被特定怪物杀死 |
| `brewed_potion` / `construct_beacon` / `used_totem` / `cured_zombie_villager` | 一次性事件 |
| `consume_item` | 吃金苹果 / 喝牛奶 |

### §6.5 `inventory_changed` + `nbt` predicate（6 个）

法术胚胎按 `EmbryoSpellId` NBT 分 5 种：

```json
"items": [
  {
    "items": ["corpse_campus:spell_embryo"],
    "nbt": "{EmbryoSpellId:\"corpse_campus:sonic_sense\"}"
  }
]
```

## §8 维护指南

### §8.1 新增 A/S 法术成就

1. 新建 `ability/<spell_id>.json`（parent 挂对应流派的 `advance`/`summit`），trigger 用 `impossible`
2. 在 [AnomalyAdvancements.java](src/main/java/com/mifan/anomaly/AnomalyAdvancements.java) 的 `SPELL_LEARN` Map 新增一行
3. 不需要改 `AnomalyBookService`，因为 `onSpellLearned` 已在 `addSpell` / `applyAwakening` / 卷轴通道三处自动调用

### §8.2 stats 阈值单位陷阱

- `damage_dealt` / `damage_taken` 是 **1/10 点伤害**："500 点" → `5000`
- `*_one_cm` 是 **厘米**："200 格" → `20000`
- `sneak_time` / `play_time` 是 **tick**：20 tick = 1 秒

### §8.3 文字与图标

- 所有 `title`/`description` 硬编码中文。英文客户端也显示中文（比 raw key 好看）。
- 图标用原版 MC 物品。如果未来想换自制图标，改 `display.icon.item` 即可。

### §8.4 Hook 与幂等性

`PlayerAdvancements.award` 是幂等的——成就拿过后重复调用无副作用。因此 `ensureBookPresent` 末尾每次都调 `onBookGranted` 是安全的。

### §8.5 已知遗漏

1. **待接入 Hook 5 个**（§5）— 对应 `natural/dungeon`、`natural/danger_sense_crawl`、`finale/endless_life_cast`、`finale/rewind_mirror`、`finale/grafter_cross`。
2. `natural/mark.json` 用 `placed_block` trigger + `used oak_sign` stat 组合，**如果 MC 放置告示牌不累加 used stat**，此成就不触发。需要实机验证。
3. `exploration/end.json` 的 `from` 字段留空，意为从任意维度穿越到末地。若 MC 解析严格可能失败，可加默认 overworld 或同时覆盖 nether→end。
4. 中文标题 hidden 隐藏后，玩家看到"??? 未知"；达成后显示真标题——MC 原生行为。
