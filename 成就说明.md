**说明**：
- `damage_dealt` / `damage_taken` 的单位是 **1/10 点伤害**（MC 原版惯例），所以 "500 点" → 阈值写 `5000`，"200 点" → `2000`。
- `wanxiang` 用 **累加** `fall_one_cm` 近似原设定「一次性坠落 200 格」，语义有偏差。
- `magnetic_cling` 用 `climb_one_cm ≥ 500`（5 米累计）近似 "贴墙 5 分钟"。

**依据**：[自然觉醒.md §3.3 条件原子集合](./自然觉醒.md) + MC 1.20.1 `net.minecraft.stats.Stats` 类名单 + 各 `natural/*.json` 现场字段。

## §4 触发器机制分类解说

### §4.1 `minecraft:inventory_changed`（最常用）

**语义**：玩家背包发生任何变动（含装备 / 主副手 / 快捷栏），且**当前持有**列表中指定物品。

```json
"trigger": "minecraft:inventory_changed",
"conditions": {
  "items": [{"items": ["corpse_campus:trait_xujing_b"]}]
}
```

**行为细节**：
- **"曾经持有过"即算达成**：物品被丢弃后，成就不会回撤。
- 多 `items` 数组元素之间是 **AND**（玩家需同时持有所有列出物品）——所以 `all_five` 的五条是分开的 criteria，用 `requirements` AND 组合，而不是一个 items 数组里塞 5 个物品。

### §4.2 `minecraft:tick` + `stats`（累计触发）

**语义**：玩家每 tick 被检查一次，当 `stats` 里列出的数值全部满足时触发。

```json
"trigger": "minecraft:tick",
"conditions": {
  "player": [{
    "condition": "minecraft:entity_properties",
    "entity": "this",
    "predicate": {
      "type_specific": {
        "type": "minecraft:player",
        "stats": [{
          "type": "minecraft:custom",
          "stat": "minecraft:mob_kills",
          "value": {"min": 200}
        }]
      }
    }
  }]
}
```

**关键 stat 清单**（本项目用到）：
- `minecraft:custom` + `minecraft:mob_kills` — 击杀怪物累计数
- `minecraft:custom` + `minecraft:damage_dealt` — 造成伤害（1/10 点）
- `minecraft:custom` + `minecraft:damage_taken` — 承受伤害（1/10 点）
- `minecraft:custom` + `minecraft:deaths` — 死亡次数
- `minecraft:custom` + `minecraft:fall_one_cm` — 坠落厘米数
- `minecraft:custom` + `minecraft:climb_one_cm` — 攀爬厘米数
- `minecraft:custom` + `minecraft:sneak_time` — 蹲伏 tick 数
- `minecraft:custom` + `minecraft:interact_with_crafting_table` — 与工作台交互次数
- `minecraft:used` + `<item>` — 物品使用次数（含右键放置）
- `minecraft:mined` + `<block>` — 挖方块次数

**注意事项**：
- stat 数值**永不回落**（除非 `/stats` 清零）；成就拿到后不可撤销。
- `damage_dealt` / `damage_taken` 单位是 `1/10 点伤害`，需要 **×10** 换算。

### §4.3 `minecraft:placed_block`

**语义**：玩家放置方块事件。本项目只在 `natural/mark.json` 用，且叠了 `player.stats` 过滤，保证 `used oak_sign ≥ 100` 累计达标后才触发。

### §4.4 `minecraft:impossible`（占位）

**语义**：永远不会被 MC 服务端自发触发，只能通过 `player.getAdvancements().award(advancement, criterionKey)` 或 `/advancement grant` 命令手动激活。

**本项目用途**：那些依赖模组内部状态（吞噬成功、法术装入书、镜像维度穿越、施放生生不息 …）且原版 trigger 无法识别的场景，先挂上占位 JSON，待代码层补 grant 调用。

## §7 维护指南 & 可能遗漏点

### §7.1 如果未来新增 A/S 法术

1. 在 `ability/` 下新增 `<spell_id>.json`，parent 指到对应 `advance/<school>` 或 `summit/<school>`。
2. 在 §5.1 的 `SPELL_TO_ADVANCEMENT` 表里加一行。
3. 如果法术涉及新的终局事件，可以考虑加到 `finale/` 下。

### §7.2 如果要去掉"曾经持有过即算"

当前 `inventory_changed` 的语义是"曾经持有"。若需改为"当前持有"，只能换 trigger（MC 原生不支持），实际做法是：在玩家背包更新后调用 `getAdvancements().revoke(...)` 反向回撤 —— 不建议。

### §7.3 关于 `stats` 阈值单位

- `damage_dealt` / `damage_taken` 是 **1/10 点伤害**，"500 点" → `5000`
- `*_one_cm` 是 **厘米**，200 格 = 20000
- `sneak_time` 是 **tick**，5 分钟 = 6000 tick

### §7.4 `minecraft:placed_block` 与 stats 叠加

`natural/mark.json` 用 `placed_block` 触发器 + `player.stats used oak_sign ≥ 100`。这意味着：
- 前 99 次放置：不通过（stats 还没到 100）
- 第 100 次放置：trigger 触发 + stats 通过 → 成就授予
- 第 101 次之后：已授予，无害

**注意**：如果 MC 在 "used stat" 里不统计 "放置告示牌" 这种 item-used-on-block 行为，此成就将**永远不触发**。需要在客户端实测一次以校验（本项目未做实测）。

### §7.5 国际化（i18n）

所有 `title` / `description` 当前硬编码中文字符串。英文客户端也会看到中文，不会显示 raw key。若未来要英文版：

1. 把所有 `"text": "xx"` 改回 `"translate": "advancement.corpse_campus.xx.title"`
2. 在 `lang/zh_cn.json` 与 `lang/en_us.json` 中补对应键
3. 迁移动作会扫过 **73 个文件**，建议做个 script 批量替换

### §7.6 图标物品选择

当前 L2 等成就用原版 MC 物品作图标（告示牌 / 骨粉 / 铁锄…）。若之后想换成自制图标，可新增 `corpse_campus:natural_mark_icon` 等物品，替换 `display.icon.item` 即可，不需要动触发器。

### §7.7 其他可能遗漏

1. 本文所有"impossible → 代码激活"方案均未实装，当前 19 个占位成就在游戏里**永远无法达成**（只能用 `/advancement grant`）。
2. `natural/all.json` 含 2 条 impossible 子条件（crawl / dungeon），**完整通关必须走命令**，除非代码层补上。
3. `finale/first_death` 走 `deaths ≥ 1` stat，意味着"玩家死过一次"就触发，与"作为异能者死亡"语义不完全等价——异能身份判定需要代码增强。
4. 目前**没有做 "成就奖励 rewards"**（用户明确无奖励）。若以后想加 loot / function 奖励，每个 JSON 补 `"rewards": {...}` 字段即可。
5. 中文标题 hidden 隐藏后，在《成就》面板里显示为 "??? 未知"；玩家达成后才显示真标题 —— 这是 MC 原生行为，属于预期。
