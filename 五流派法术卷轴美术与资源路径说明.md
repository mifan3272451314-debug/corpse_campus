# 五流派象征物美术与资源路径说明

## 1. 现在不再做“卷轴”，而是做“流派象征物”

如果这五个流派不想继续使用统一卷轴造型，而是改成更有辨识度的**象征物 / 信物 / 焦点物 / 法器**，这是更适合当前世界观的方向。

相比“同模板换颜色的卷轴”，象征物方案的优势更明显：

1. **辨识度更高**：玩家一眼就能区分五个流派
2. **更贴设定**：虚境、日兆、东岳、愚者、圣祈本身就有强烈象征意味
3. **更适合后续扩展**：可以继续用于法术书封面、UI 图标、职业徽记、材料道具、祭器设计
4. **代码层面不受影响**：本质仍然只是一个物品贴图与物品模型，外观是卷轴还是眼睛、手、刀、假面、圣兽，都不影响注册逻辑

---

## 2. 资源路径仍然放在物品目录

即使不做卷轴，只要它们仍然是“物品”，最合适的路径依然是：

- [`corpse_campus/src/main/resources/assets/corpse_campus/textures/item/`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/)

原因：

1. 这类资源本质上是**物品图标**
2. 不属于技能按钮图标，所以不建议放进 [`textures/spells/`](corpse_campus/src/main/resources/assets/corpse_campus/textures/spells/)
3. 后续物品模型 JSON 仍然走 Minecraft 常规物品资源路径

## 3. 关于文件名：可以先沿用现有 `*_scroll` 命名

如果你已经在项目里用了以下物品名和模型名：

- [`xujing_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/xujing_scroll.png)
- [`rizhao_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/rizhao_scroll.png)
- [`dongyue_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/dongyue_scroll.png)
- [`yuzhe_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/yuzhe_scroll.png)
- [`shengqi_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/shengqi_scroll.png)

那么**现在可以只改美术概念，不急着改内部资源名**。

也就是说：

- 资源文件名依旧可以保持 `xujing_scroll.png`
- 但图画内容改成“虚境之眼”
- 这样可以避免你连带修改已有模型 JSON、注册名、语言 key、数据标签等内容

也就是：

- **外在设计改为象征物**
- **内部资源名暂时保留 scroll**
- 等以后体系稳定，再统一重构命名也不迟

如果你希望现在就完全改名，也可以改成：

- `xujing_relic.png`
- `rizhao_relic.png`
- `dongyue_relic.png`
- `yuzhe_relic.png`
- `shengqi_relic.png`

但从工程效率看，**先不动内部文件名更稳**。

## 4. 五流派象征物推荐对应

结合五个流派的定位，推荐采用以下一套：

1. **虚境 → 眼**
2. **日兆 → 手**
3. **东岳 → 刀 / 骨刃**
4. **愚者 → 假面 / 操偶之手**
5. **圣祈 → 圣心 / 灵魂圣器**

这是当前最稳、最容易拉开差异的一套。

如果你希望尽量贴近你前面提到的“眼睛、手、刀、剑、野兽”，可以进一步细化为：

- **虚境：观测之眼**
- **日兆：托日之手**
- **东岳：冥岳骨刃**
- **愚者：欺诈假面**
- **圣祈：圣祈之心**

## 6. 五流派统一风格建议

即便改成五种不同象征物，仍然建议保持以下统一点：

1. **同一绘制语言**
   - 都是像素风 / 同一批次手绘风
   - 都采用中心构图
   - 都是单个主体

2. **同一图标规则**
   - 同尺寸，例如 `32x32`、`64x64`
   - 主体居中
   - 外轮廓清晰
   - 不要背景过满

3. **同一世界观工艺感**
   - 都像“古老流派遗物”
   - 都可以带一点法术能量与刻印
   - 不要做成完全不同文明的五套风格

4. **同一使用场景适配**
   - 既能当物品图标
   - 也能裁成 UI 图标
   - 也能延展到流派徽记、法术书封面符号

这样玩家会感觉：

1. 它们属于同一世界观体系
2. 但每个流派又有完全不同的精神象征

## 9. 最终落地建议

如果你现在就要开始做图，推荐按以下原则直接执行：

### 方案 A：不改内部资源名，只改图内容
继续保留：

- [`xujing_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/xujing_scroll.png)
- [`rizhao_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/rizhao_scroll.png)
- [`dongyue_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/dongyue_scroll.png)
- [`yuzhe_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/yuzhe_scroll.png)
- [`shengqi_scroll.png`](corpse_campus/src/main/resources/assets/corpse_campus/textures/item/shengqi_scroll.png)

但图像内容分别画成：

- 虚境之眼
- 托日之手
- 冥岳骨刃
- 欺诈假面
- 圣祈之心

这是当前最省事、最稳的方案。

### 方案 B：连内部命名一起改成 relic / sigil / focus
如果你后面要整体重构资源命名，再统一改也可以，但这会涉及：

- 贴图名
- `models/item/*.json`
- 语言 key
- 注册名
- 可能的 tag / loot / recipe / datagen

所以建议先做方案 A。

这五个物品不再是“法术卷轴”，而是五个流派的**本源象征物**：
